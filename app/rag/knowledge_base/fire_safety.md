# RoboVision-Agent Knowledge Base — Fire Safety & Alarm Handling

---

## 1. 报警分级与触发条件

RoboVision-Agent 通过 `app/runtime/unified_pipeline.py` 对视频帧进行 YOLO 抽帧检测，当检测到 `fire` 或 `smoke` 类别时，调用 `fire_alarm_rules.py` 按以下规则判定报警等级。

### 1.1 HIGH 报警（红色）

| 项目 | 说明 |
|------|------|
| 触发条件 | fire 置信度 >= 0.5 连续出现 3 帧；或 fire 与 smoke 同时出现 |
| 置信度阈值 | 0.5 |
| 连续帧数 | 3 |
| 冷却期 | 同一区域 60 秒内不重复触发 |

**处置流程**：
1. 立即通知现场安全员
2. 触发声光报警器
3. 拨打 119 火警电话
4. 启动人员疏散程序
5. 切断非消防电源
6. 系统自动保存报警截图至 `data/alarms/fire/`，写入 `data/logs/fire_alarm_log.csv`

**人工复核**：安全员须在 30 秒内确认为真实火情或误报。

### 1.2 MEDIUM 预警（橙色）

| 项目 | 说明 |
|------|------|
| 触发条件 | smoke 置信度 >= 0.4 连续出现 10 帧 |
| 置信度阈值 | 0.4 |
| 连续帧数 | 10 |
| 冷却期 | 同一区域 30 秒 |

**处置流程**：
1. 通知巡检人员前往确认
2. 检查是否有可燃物堆积
3. 确认通风系统是否正常
4. 如确认为火情前兆，手动升级为 HIGH 报警

**人工复核**：巡检人员在 2 分钟内到达现场。

### 1.3 LOW 提示（蓝色）

| 项目 | 说明 |
|------|------|
| 触发条件 | 低于上述阈值的 fire/smoke 检测 |
| 处置 | 仅记录日志，不触发报警，作为后续巡检参考 |

---

## 2. 报警冷却机制

- HIGH 报警冷却期：60 秒
- MEDIUM 预警冷却期：30 秒
- 目的：避免同一事件重复报警造成告警疲劳

---

## 3. 与 RoboVision-Agent 系统联动

| 系统模块 | 功能 |
|----------|------|
| `unified_pipeline.py` | 图片/视频抽帧检测，调用 YOLO 模型 |
| `fire_alarm_rules.py` | 连续帧确认 + 冷却期判定 |
| `fire_log_tool.py` | 查询报警记录、按等级统计 |
| `event_logger.py` | 统一事件日志写入 `event_log.csv` |
| `main.py` Gradio 控制台 | 前端展示报警截图 + Markdown 摘要 |

---

## 4. 消防安全规范

- 所有检测区域须安装烟雾探测器
- 消防通道必须保持畅通
- 灭火器每月检查一次
- 每季度进行一次消防演练
- 报警记录保存至少 90 天
